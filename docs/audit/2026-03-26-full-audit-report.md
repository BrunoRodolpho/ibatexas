## Cross-Domain Insights

### 1. Security × Testing Overlap
The `security-auditor`'s P0 findings and `test-strategist`'s high-risk untested paths converge:
- **Reservation TOCTOU** (test gap) is also a security race condition
- **Customer LGPD export** (untested) could leak PII if incomplete
- **Order ownership guards** (untested) are IDOR protection — security-critical
- **Admin auth bypass** (security P0) + **0% admin test coverage** (test gap) = double blind spot

**Recommendation:** When writing domain service tests (8h from test-strategist), incorporate security assertions (ownership checks, auth guards, rate limits).

### 2. Frontend × Testing Overlap
Both `ui-designer` and `test-strategist` independently flagged:
- **0% UI package coverage** — 16 atoms, 13 organisms completely untested
- **0% admin coverage** — 11 pages with data tables, filters, mutations
- **Missing error boundaries** — checkout page lacks fallback UI

**Recommendation:** Start with atom snapshot tests (15 min), then admin page smoke tests (3h). Both agents agree this is the highest-ROI frontend investment.

### 3. Architecture × Security Overlap
`code-reviewer`'s architecture analysis and `security-auditor`'s auth findings converge:
- **Scattered authorization** (8+ files with auth checks) makes it easy to miss a route
- **No centralized policy engine** means new endpoints can ship without auth
- `feature-architect`'s ARCH-001 plan directly addresses this with a policy-based system

**Recommendation:** ARCH-001 Phase 1 (extract auth middleware) should be prioritized alongside security P0 fixes.

### 4. Code Quality × Feature Planning Overlap
`code-reviewer`'s error handling finding (P1.2) directly feeds `feature-architect`'s OBS-001 plan:
- Inconsistent error shapes → inconsistent Sentry reporting → incomplete observability
- Standardized errors + OpenTelemetry traces = comprehensive debugging

**Recommendation:** Fix error handling patterns (2h) before adding distributed tracing — the traces are only as good as the errors they capture.

### A. Security Audit (6.8/10 — NO-GO → 10h remediation)

**P0 Critical (5 findings):**

| # | Finding | File | Line | Fix |
|---|---------|------|------|-----|
| CRIT-001 | **Secrets committed to Git** — .env with Twilio, Anthropic, JWT keys in repo | `.env` | 2, 51-56, 63-64 | Rotate ALL secrets + BFG scrub history (2h) |
| CRIT-002 | **Hardcoded JWT fallback** — `"test-secret-do-not-use-in-production"` | `apps/api/src/server.ts` | 58 | Remove fallback, throw if missing (5m) |
| CRIT-003 | **Admin API key in browser bundle** — `NEXT_PUBLIC_ADMIN_API_KEY` compiled into JS | `apps/admin/src/lib/api.ts` | 17 | Remove prefix, use JWT cookie auth only (1h) |
| CRIT-004 | **Admin auth disabled in non-prod** — `NODE_ENV !== "production"` bypass | `apps/admin/src/middleware.ts` | 29-32 | Remove bypass, enforce auth everywhere (20m) |
| CRIT-005 | **Missing secret validation** — no min length/entropy check at startup | `apps/api/src/server.ts` | 53-60 | Add `requireSecret()` with 32-byte minimum (30m) |

**P1 High Priority (6 findings):**

| # | Finding | File | Fix |
|---|---------|------|-----|
| HIGH-002 | WhatsApp webhook URL misconfiguration risk | `whatsapp-webhook.ts:139` | Add startup validation + HTTPS enforcement (20m) |
| HIGH-003 | Admin API key DoS — no size limit on x-admin-key header | `admin/index.ts:62-68` | Add 256-byte max header check (30m) |

| HIGH-006 | OTP rate limiting insufficient — 18 code attempts possible per window | `auth.ts:60-72` | Tighten to 3 verify attempts per 5min + exponential backoff (1h) |
